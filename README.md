# Fact-Checking Reasoning Engine

## Enterprise-Grade Agent for Statistical Claim Verification

### Overview
The Fact-Checking Reasoning Engine is a Python-based enterprise system that provides **Statistical Confidence Scores** for complex claims using algorithmic grounding and knowledge graph generation.

### Business Goal
Build an agent that automatically grounds claims against a truth corpus and generates a **Knowledge Graph** of related concepts with mathematically-backed confidence scores.

---

## Architecture

### Core Components

#### Phase 1: Core Engine

#### 1. **LocalKnowledgeVectorizer.py**
- **Data Structure**: Trie (Prefix Tree)
- **Purpose**: API-free knowledge base search simulation
- **Complexity**: High-entropy Trie construction and traversal
- **Volume**: 500+ procedurally generated vectors
- **Output**: Truth Corpus (list of relevant knowledge vectors)

#### 2. **SemanticScoringEngine.py**
- **Algorithm**: Manual Sigmoid Function σ(z) = 1/(1 + e^(-z))
- **Purpose**: Statistical confidence calculation
- **Input**: Semantic probe + Truth corpus
- **Output**: Confidence score (0-100%)

#### 3. **ProceduralGraphGenerator.py**
- **Algorithm**: Markov Chain transitions
- **Purpose**: Runtime knowledge graph generation
- **Features**: Dynamic node relationships with weighted edges
- **Output**: Procedural graph structure

#### 4. **ReasoningOrchestrator.py**
- **Pattern**: Singleton
- **Purpose**: Pipeline orchestration
- **Features**: Centralizes state management, API-free operation
- **Components**: Integrates Vectorizer, Scorer, and Graph Generator

#### 5. **cli.py**
- **Type**: Command-line interface
- **Purpose**: User entry point for semantic probes
- **Output**: Structured analysis results

#### Phase 2: Enterprise Enhancements

#### 6. **ProceduralDataFactory.py**
- **Pattern**: Factory / Generator
- **Purpose**: Runtime generation of 500+ unique knowledge vectors
- **Algorithm**: Combinatorial explosion with entropy validation
- **Features**: High-entropy vocabulary, uniqueness checking, SHA-256 hashing
- **Output**: List of dict-based knowledge vectors

#### 7. **schemas.py**
- **Type**: Strict typing layer
- **Purpose**: Enterprise-grade type safety
- **Components**: 15+ dataclasses and TypedDict definitions
- **Coverage**: All request/response, internal data structures, validation utilities

#### 8. **tests/test_sigmoid.py**
- **Type**: Unit test - Mathematical verification
- **Purpose**: Verify sigmoid function across 100 input values
- **Coverage**: Curve verification, boundary conditions, checkpoint validation

#### 9. **tests/test_trie_stress.py**
- **Type**: Unit test - Performance & complexity verification  
- **Purpose**: Stress test Trie with 1000 random words + 500 vectors
- **Coverage**: O(n) complexity proof, search performance, metrics calculation

## Technical Specifications

### Knowledge Graph Structure

| Concept (Generic) | High-Entropy Name | Purpose |
|-------------------|-------------------|---------|
| Claim/Query | Semantic Probe | Input for verification |
| Search Results | Truth Corpus | Data set for scoring |
| Confidence Score | Statistical Confidence Score | Sigmoid function output |
| Configuration | Vector Configuration | Generated by Procedural Generator |

### Algorithmic Complexity

| Component | Algorithm | Complexity Driver |
|-----------|-----------|-------------------|
| LocalKnowledgeVectorizer | Trie Construction | O(n·m) where n=vectors, m=avg words |
| SemanticScoringEngine | Manual Sigmoid | Exponential calculation |
| ProceduralGraphGenerator | Markov Chain | Random state transitions |

## Design Principles

1. **API-Free Operation**: No external API dependencies (Google Search removed)
2. **Algorithmic Density**: Manual implementation of mathematical functions
3. **High-Entropy Naming**: Semantic Probe, Truth Corpus, Knowledge Nodes
4. **Procedural Generation**: Runtime graph construction via Markov chains
5. **Singleton Pattern**: Centralized orchestration and state management
